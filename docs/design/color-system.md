# 配色方案

## 目标

工作台的整体气质应更接近干净、克制的企业 SaaS 产品，而不是高饱和仪表盘，也不是大面积蓝色界面。

蓝色只作为强调色存在。

基础视觉语言应建立在以下几种颜色之上：

- 白色
- 极浅灰
- 极浅边框灰

参考值：

- `#F8FAFC`
- `#F5F7FA`
- `#EEF2F6`

## 配色策略

### 基础表面色

以下颜色应作为默认背景系统：

- 页面背景：接近白色
- 卡片背景：纯白
- 次级区块背景：`#F8FAFC`
- 更弱一级或 muted 背景：`#F5F7FA`
- 边框：`#EEF2F6`

### 文本颜色

大部分正文应优先使用中性深色文本，而不是蓝色文本。

推荐层级：

- 主文本：深中性 slate
- 次文本：偏灰的 slate
- 元信息文本：更浅一级的中性灰

只有在确实承担强调或动作提示时，才使用蓝色文本。

### 蓝色强调

蓝色适合保留在：

- 图标底块
- 当前导航激活态
- 数字强调
- CTA 强调
- 箭头和方向提示
- focus 或交互高亮

蓝色不适合用于：

- 整体布局背景
- 多个相邻大面板
- 默认的整段大块内容背景
- 同一屏中反复堆叠的蓝色浅底区块

## 当前 Token 方向

当前 `apps/web/src/app/globals.css` 中的实现，遵循的是以下语义拆分：

- `background`：接近白色的工作台背景
- `card`：白色主卡片
- `secondary`：浅中性灰表面
- `accent`：带一点轻微倾向的辅助表面
- `border`：极浅分隔线与卡片轮廓
- `primary`：只承担蓝色强调

后续即便调整具体颜色值，也应尽量保留这层语义关系，不要把 `primary` 再重新扩散成主背景色。

## 组件级规则

### 工作台首页

- 大容器应以白色或极浅灰为主
- 主操作卡应整体偏中性，蓝色只用于图标和动作强调
- 协作快照区块不应再回到深蓝或高饱和蓝底，除非整体设计方向明确改变

### 侧边栏与顶部栏

- 壳层背景应保持接近白色
- 蓝色只用于当前项高亮，不作为常驻填充色
- 选中态优先使用浅蓝底加深色文字，不要回到高对比蓝底加白字

### 编辑器、RAG 检索页与设置页

文档页 AI 已内嵌在编辑器中，不再使用右侧独立 AI 面板。`/search` 检索工作台与设置页仍可能与文档编辑页共享同一套配色。为了避免整页发白或泛蓝：

- 白色卡片应是主层
- 内部辅助区块优先使用 `#F8FAFC` 或 `#F5F7FA`
- citation 区、辅助说明块、metadata 芯片要尽量克制
- 没有功能必要时，不要并排堆多个带蓝色底的区块

## 反模式

应避免以下模式：

- 大面积蓝色 hero 背景
- 默认用深海军蓝做数据卡主底色
- 每一张卡片都带浅蓝底
- 长段正文使用蓝色文本
- 粗边框或高对比描边

如果蓝色重新变成主填充色，整体很容易滑向“医院管理系统感”。

## 实用检查清单

后续接受 UI 改动前，可以快速检查：

- 页面是否仍然以白色和浅灰色为主？
- 蓝色是否只出现在真正需要关注的位置？
- 相邻面板是否足够平静，而不是全部带浅蓝底？
- 边框是否足够轻，只负责分隔而不形成噪音？
- 当编辑器内嵌 AI、RAG 检索页与文档编辑页同时演进时，整页是否依然平衡？

如果以上任一问题答案是否定的，优先减少蓝色，而不是继续增加装饰。
